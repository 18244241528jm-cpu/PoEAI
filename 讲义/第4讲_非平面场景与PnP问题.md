## 先想清楚：这讲到底在解什么问题？

前三讲我们一直在处理"平面"上的点——April Tag、标定板、地面。但现实世界不是平面的：一辆汽车有高有低、一座建筑有前有后。当 3D 点不再都在同一个平面上时，单应 $H$ 就不能用了。那怎么从"散落在空间各处的 3D 点 + 它们在图像上的投影"来求相机位姿？这就是 **PnP（Perspective-N-Points）** 问题——本课程后半段的核心。

## 1. 非平面怎么办？（What if we have non-planar scenes?）

> 📖 **对照 PPT**：打开 03_single_view_pose_estimation.pdf **第 19 页** "What if we have non-planar scenes?"，这一页标志着从平面单应到 PnP 的转折。

前几讲在**平面**上建立了"平面 → 图像"的**单应** $H$，并从 $H$ 分解出 $(R, \mathbf{t})$。单应的前提是：**所有 3D 点都在同一平面**（如 $Z=0$）。

一旦场景**不是平面**——例如物体有厚度、多个不同深度的点、或一般 3D 结构——则：

- "平面上的 2D 点"与"图像 2D 点"之间**不再**是同一个 3×3 单应（不同深度对应不同的透视缩放）。
- 必须显式使用**每个点的 3D 坐标**和**相机投影模型**来求位姿。

这类问题就是 **Perspective-N-Points（PnP）**：已知若干** 3D 点**及其在图像上的** 2D 观测**，求相机的 $(R, \mathbf{t})$。

---

## 2. Perspective-N-Points（PnP）问题定义

> 📖 **对照 PPT**：打开 03_single_view_pose_estimation.pdf **第 20 页** "Perspective N points (PnP)"，列出了已知量（3D 点+2D 观测+K）和未知量（R,t）。

### 2.1 已知与未知

**已知**：

- $N$ 个 3D 点在世界坐标系下的坐标：$\mathbf{X}_1, \ldots, \mathbf{X}_N \in \mathbb{R}^3$（或齐次 $\mathbf{X}_i = (X_i, Y_i, Z_i, 1)^\top$）。
- 它们在图像上的 2D 观测（像素）：$\mathbf{x}_1, \ldots, \mathbf{x}_N$，满足 $\mathbf{x}_i \sim K(R \mathbf{X}_i + \mathbf{t})$。
- 相机内参矩阵 $K$（已标定）。

**未知**：

- 相机外参：**旋转 $R$** 与 **平移 $\mathbf{t}$**（**world-to-camera**：把世界坐标变到相机坐标）。

这就是 **PnP**：用"若干 3D 点 + 其 2D 像点"求相机位姿。$N$ 通常 $\ge 3$；后面会说明为何**至少需要 3 个点**。

### 2.2 数学形式

针孔模型（齐次坐标）：

$$
\mathbf{x} \sim K \, [R \mid \mathbf{t}] \, \mathbf{X}
$$

其中 $\mathbf{X} = (X, Y, Z, 1)^\top$ 为世界坐标，$\mathbf{x} = (u, v, 1)^\top$ 为图像齐次坐标。展开：

$$
\mathbf{x} \sim K \big( R \mathbf{X} + \mathbf{t} \big), \qquad R \in SO(3),\ \mathbf{t} \in \mathbb{R}^3
$$

PnP 就是：给定多组 $(\mathbf{X}_i, \mathbf{x}_i)$ 与 $K$，求 $(R, \mathbf{t})$。

**🧭 问题定义清楚了：PnP = 从 3D-2D 对应求 $(R, \mathbf{t})$。但为什么至少要 3 个点？下面用一个 2D 灯塔的类比来建立直觉。**

## 3. 方位角问题（Bearing Problem）：直觉从哪来

> 📖 **对照 PPT**：打开 03_single_view_pose_estimation.pdf **第 21–23 页** "Bearing Problem" / "Bearing problem: lighthouse" / "Two points is not enough"，有灯塔类比的示意图：两个灯塔无法定位、圆周角定理（inscribed angle theorem）、第三个灯塔消除歧义。**这三张图是本讲最重要的视觉辅助，务必打开看。**

### 3.1 灯塔类比（2D）

想象你在海上，只知道**两个灯塔相对于你的方向**（方位角），但**不知道到灯塔的距离**：

- 每个"方向"对应一条从你出发的**射线**。
- 你只知道两条射线的夹角，不知道你在这两条射线上的位置。
- **两条射线**在平面上会交于一点吗？不会——它们从"你"出发，你可以在两条射线**交点**（即你的位置）沿射线**前后移动**，方向不变，所以**仅凭两个方向无法确定你的位置**。

换句话说：**只知道两个 bearing（方向），无法唯一确定自己的 2D 位置**。

### 3.2 一个具体的数字例子

假设你在坐标 $(x, y)$ 处（未知），看到灯塔 A 在正北方向，灯塔 B 在东偏北 $45°$ 方向。已知灯塔 A 的真实坐标是 $(0, 10)$，灯塔 B 的真实坐标是 $(5, 5)$。仅凭方向你能确定自己的位置吗？

- "正北方向看到 A" 意味着你在 A 的正南方，即 $x = 0$、$y < 10$。你可以在 $(0, 0)$，也可以在 $(0, 5)$、$(0, -100)$……沿这条南北射线上任何位置都满足条件。
- 对应每个候选位置 $(0, y_0)$，"东偏北 $45°$ 方向看到 B" 要求 $(5 - 0, 5 - y_0) \parallel (1, 1)$，即 $5 = 5 - y_0$，所以 $y_0 = 0$。

看起来这里恰好解出了 $(0, 0)$？——但这是因为我们同时用了**方向 + 已知灯塔绝对坐标**。在纯 bearing 问题（只知道灯塔间的**相对角度**而不知道绝对方向参考）中，你知道的只是"A 和 B 之间的夹角是 $45°$"。由圆周角定理，满足"对 A–B 张角 $45°$"的点构成一段**圆弧**——不是一个点，而是一整段弧！这就是**两个灯塔不够**的根本原因：解是一条曲线而非一个点。

### 3.3 第三个灯塔的作用（圆周角定理）

若再知道**第三个灯塔的方向**：

- 你、灯塔 1、灯塔 2 的相对方向确定了一个**角度**（例如角 1–你–2）。
- **圆周角定理（inscribed angle theorem）**：在平面上，到两定点张角为定值的点，落在某段**圆弧**上。
- 两个这样的圆弧（例如"对灯塔 1–2 张角为 $\alpha$"的弧 与 "对灯塔 2–3 张角为 $\beta$"的弧）一般会交于**有限个点**（常为 1 个或 2 个）。
- 因此**第三个方向**能帮助把位置确定下来（可能仍有离散歧义，但不再是"整条射线"上的自由度）。

所以直观上：**至少需要 3 个已知方向的观测**，才能从"纯方向"信息中恢复位置。对应到 PnP：**至少需要 3 个 3D 点**（每个点给出一条从相机出发的射线方向）。

## 4. 在 3D 中的对应关系（Similar in 3D）

> 📖 **对照 PPT**：打开 03_single_view_pose_estimation.pdf **第 24 页** "Similar in 3D"，有从相机光心出发的射线穿过图像平面到 3D 点的示意图。

### 4.1 像素 = 相机系下的一条射线

在相机模型里：

- 图像上的一个**像素** $(u, v)$ 对应相机坐标系下的一条**射线**：从光心出发、穿过该像素的 3D 方向。
- 归一化平面上的点 $\mathbf{x}_{\text{norm}} = K^{-1} (u, v, 1)^\top$ 即该射线的**方向向量**（可再单位化）。
- 你**不知道**这条射线上的 3D 点离相机多远——只知道**方向**（bearing）。

所以：每个 2D 观测等价于"在相机系下给出一条射线方向"；若该观测对应世界系中的某个 3D 点 $\mathbf{X}$，则**该 3D 点一定在这条射线上的某处**，即存在深度 $\lambda > 0$ 使得（在相机系下）$\mathbf{X}_c = \lambda \cdot \text{direction}$。

### 4.2 为何 3D 里也要至少 3 个点

- **1 个点**：只给一条射线，相机可以在这条射线上任意移动（深度未知）→ 位姿不可定。
- **2 个点**：两条射线方向已知；相机可以在"使这两个 3D 点分别落在这两条射线上"的约束下，仍有**绕两射线所在平面的旋转等自由度**（类似 2D 里"两个灯塔"不够），无法唯一确定位姿。
- **3 个点**：三条射线 + 三个 3D 点之间的已知几何关系，一般可把相机位姿确定到**有限组解**（如 P3P 最多 4 组）；再加**第 4 个点**可区分这些解，得到唯一位姿。

因此：**PnP 最少需要 3 个点**（最小解称为 P3P）；**唯一性**通常需要至少 4 个点（或 3 个点加额外约束/选解策略）。

## 5. PnP 的数学形式再写一遍

> 📖 **对照 PPT**：打开 03_single_view_pose_estimation.pdf **第 25–26 页** "Perspective-N-Points (PnP)"，写出了完整的数学形式和 OpenCV solvePnP 链接。

设世界系下 3D 点为 $\mathbf{X} = (X, Y, Z, 1)^\top$，相机系下为 $\mathbf{X}_c = R\mathbf{X} + \mathbf{t}$（这里把 $\mathbf{X}$ 视为 $(X,Y,Z)^\top$ 时即 $R\mathbf{X} + \mathbf{t}$）。投影到图像：

$$
\mathbf{x} \sim K \mathbf{X}_c = K \big( R \mathbf{X} + \mathbf{t} \big)
$$

或写成投影矩阵形式 $P = K[R\mid\mathbf{t}]$，则 $\mathbf{x} \sim P\mathbf{X}$。

**PnP 问题**：给定 $(\mathbf{X}_i, \mathbf{x}_i)$，$i=1..N$，$N\ge 3$，与 $K$，求 $(R, \mathbf{t})$ 使得对每个 $i$ 有 $\mathbf{x}_i \sim K(R\mathbf{X}_i + \mathbf{t})$。

---

## 6. 与 Homography（平面）的对比

| 方面 | 平面单应（Homography） | PnP |
|------|------------------------|-----|
| **场景** | 所有点在同一平面（如 $Z=0$） | 任意 3D 点，可不同深度 |
| **几何** | 平面 → 图像 = 2D 射影变换 $H$ | 每个点独立满足 $\mathbf{x} \sim K(R\mathbf{X}+\mathbf{t})$ |
| **未知量** | 3×3 矩阵 $H$（8 dof） | $(R, \mathbf{t})$（6 dof） |
| **最少点数** | 4 对（平面 2D ↔ 图像 2D） | 3 对（3D ↔ 2D）可解；4 对更稳、可唯一 |
| **典型用途** | April Tag、标定板、已知平面 | 稀疏 3D 点 + 特征匹配、SLAM、物体位姿 |

当场景**退化为平面**时，PnP 仍适用（相当于所有点 $Z=0$）；但平面情况下用单应 $H$ 再分解往往更简单、更稳定。**非平面**则必须用 PnP（或等价形式）。

## 7. OpenCV 与实现

OpenCV 提供 **`cv2.solvePnP`**（及带 RANSAC 的 `cv2.solvePnPRansac`）：

- **输入**：世界系下 3D 点数组、对应图像 2D 点数组、内参 $K$、畸变系数等。
- **输出**：旋转向量（通常为 axis-angle）、平移向量 $\mathbf{t}$（world-to-camera）。
- 内部会使用**非线性优化**（最小化重投影误差）或**最小解 + RANSAC**（如 P3P）等，下一讲和第六讲会分别讲这两类方法。

文档示例：  
[https://docs.opencv.org/4.x/d5/d1f/calib3d_solvePnP.html](https://docs.opencv.org/4.x/d5/d1f/calib3d_solvePnP.html)

## 8. 本讲小结与"学完你能做什么"

| 内容 | 要点 |
|------|------|
| **非平面** | 单应只适用于平面；一般 3D 场景用 PnP。 |
| **PnP 定义** | 已知 $N$ 个 3D 点（世界坐标）及其 2D 观测（像素）与 $K$，求 $(R, \mathbf{t})$（world-to-camera）。 |
| **Bearing 问题** | 仅知道"方向"不知道距离时，2D 上两个灯塔无法定位置。 |
| **至少 3 个点** | 圆周角定理、第三个方向消除歧义；3D 中像素=射线，三条射线+几何约束可定位姿（有限组解）。 |
